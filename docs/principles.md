# Principles

**Version:** 1.1  
**Date:** 2026-03-30  
**Status:** Supporting doc

This repository has a strong default path, but it is not trying to standardize every project into the same shape.

## Core Beliefs

- The method should stay useful even if the preferred AI vendor changes.
- Documentation should help humans understand the project, not just help AI continue a session.
- Templates should reduce blank-page friction without forcing cargo-cult structure.
- Examples should teach practice, not become hidden requirements.
- Vendor-specific resources should be explicit and removable.

## Constraints

- Core documents must remain vendor-neutral.
- Vendor instructions must live in vendor pack directories.
- Reusable templates must avoid references to specific platforms or products.
- Example artifacts should be clearly labeled as examples, not canonical required forms.

## Non-Goals

- This repo is not a product roadmap for software to be shipped.
- This repo is not a prompt collection with no surrounding method.
- This repo is not an attempt to automate judgment out of the process.
- This repo is not a universal standard every project must copy exactly.

## Default Posture

The default posture is:

- opinionated enough to be actionable
- flexible enough to adapt to different projects
- explicit about what is core versus optional
